# Workflow Script Review Checklist

Purpose: review behavior by script file number so editing and scenario validation stay simple.

## How to Use

1. Pick one script block by number.
2. Add scenario cases you want to validate.
3. Compare current behavior with the workflow outline.
4. Decide one of: keep as-is, adjust script, adjust outline.
5. Record the test command and result.

## Script Map

1. 01_sync_work.py = starts the sync step
2. 02_promote.py = starts the promote step
3. 03_package.py = starts the package step
4. 04_run_chain.py = runs all numbered steps in order
5. 05_pipeline_engine.py = the main logic file that does the real work
